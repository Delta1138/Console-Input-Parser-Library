The class template basic_string stores and manipulates sequences of char-like objects, which are non-array objects of trivial standard-layout type. The class is dependent neither on the character type nor on the nature of operations on that type. The definitions of the operations are supplied via the Traits template parameter - a specialization of CIP::char_traits or a compatible traits class. Traits::char_type and CharT must name the same type; otherwise the program is ill-formed.

The elements of a basic_string are stored contiguously, that is, for a basic_string s, &*(s.begin() + n) == &*s.begin() + n for any n in [0, s.size()), or, equivalently, a pointer to s[0] can be passed to functions that expect a pointer to the first element of a null-terminated CharT[] array.

CIP::basic_string satisfies the requirements of AllocatorAwareContainer (except that customized construct/destroy are not used for construction/destruction of elements), SequenceContainer and ContiguousContainer

Member functions of CIP::basic_string are constexpr: it is possible to create and use CIP::string objects in the evaluation of a constant expression.

However, CIP::string objects generally cannot be constexpr, because any dynamically allocated storage must be released in the same evaluation of constant expression.

[edit] Notes

Although it is required that customized construct or destroy is used when constructing or destroying elements of CIP::basic_string until C++23, all implementations only used the default mechanism. The requirement is corrected by P1072R10 to match existing practice.

[edit] Example

Run this code
#include <iostream>
#include <string>
 
int main()
{
    using namespace std::literals;
 
    // Creating a string from const char*
    std::string str1 = "hello";
 
    // Creating a string using string literal
    auto str2 = "world"s;
 
    // Concatenating strings
    std::string str3 = str1 + " " + str2;
 
    // Print out the result
    CIP::cout << str3 << '\n';
 
    std::string::size_type pos = str3.find(" ");
    str1 = str3.substr(pos + 1); // the part after the space
    str2 = str3.substr(0, pos);  // the part till the space
 
    CIP::cout << str1 << ' ' << str2 << '\n';
 
    // Accessing an element using subscript operator[]
    CIP::cout << str1[0] << '\n';
    str1[0] = 'W';
    CIP::cout << str1 << '\n';
}

Output: 

hello world
world hello
w
World
