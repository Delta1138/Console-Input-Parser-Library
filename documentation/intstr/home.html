CIP::intstr

From cppreference.com
< cpp‎ | string
struct intstr;

CIP::intstr represents a simple descision tree which operates on a int which it wraps, where an input string either throws an invalid_input or causes the wrapped int to be set to a corresponding value.
It holds three things: A int*, and two const char* arrays (which can be nullptr) When intstr::assignValue(std::string s) runs, it checks if s is equal to any of the contained strings. If it is, the wrapped int is set to the corresponding value. If not, it throws an invalid_input if it was run as part of input_handler::interpretInput(), and otherwise throws an std::runtime_error.
The primary use of this class is that it is a type accepted by input_handler, so it can be used to create a switch which sets the program's mode based on a user-input string.

[edit] Example

#include <CIP.h>
 
int main()
{
    const int create  = 0;
    const int destroy = 1;
    const int modify  = 2;
    const int invert  = 3;
    const int copy    = 4;
    const int rename  = 5;
    int mode;
    int example2;
    int example3;
 
    // Wrapping the int vars and declaring the first set of matching strings for each
    CIP::intstr modewrap(mode, create, "c", "create");
    CIP::intstr   e2wrap(example, 10, "ten");
    CIP::intstr   e3wrap(example, 999, "high");
 
    // declaring the rest of the matching strings
      e3wrap.setMarkersFor(-999, "low");
      e2wrap.setMarkersFor(11, "eleven");
      e2wrap.setMarkersFor(-1, "negone", "negOne", "neg_one");
    modewrap.setMarkersFor(destroy, "d", "destroy");
    modewrap.setMarkersFor(modify, "m", "modify");
    modewrap.setMarkersFor(invert, "i", "invert");
    modewrap.setMarkersFor(copy, "c", "copy");
    modewrap.setMarkersFor(rename, "r", "rename");
 
    // creating an input_handler which takes two command-line inputs and
    // sets the three ints according to them
    CIP::input_handler ih(argc, argv);
    CIP_ORDERED_START(ih);
    ih.setOrdered(modewrap);
    ih.setOrdered(e2wrap);
    ih.setOrdered(e3wrap);
    CIP_ORDERED_END(ih);
 
    // sets the ints based on argv
    ih.interpretInput();
}

Results: 

 mode gets set to:
  - 0 (destroy) if the first input is "d" or "destroy"
  - 1 (modify)  if the first input is "m" or "modify"
  - 2 (invert)  if the first input is "i" or "invert"
  - 3 (copy)    if the first input is "c" or "copy"
  - 4 (rename)  if the first input is "r" or "rename"

 example2 gets set to:
  - 10 if the second input is "ten"
  - 11 if the second input is "eleven"
  - -1 if the second input is "negone", "negOne" or "neg_one"
  
 example3 gets set to:
  - 999 if the third input is "high"
  - -999 if the third input is "low"
  
an invalid_input exception is thrown if any input 
does not match one of the corresponding strings
