Used to enable and set the behavior of dynamic input

1) Sets up dynamic input with a list of uncahnging prompts
2) Sets up dynamic input with a list of prompts, which can change based on past input
3) Enables dynamic input, and makes it act like it's reading input from the command line.

[edit] Parameters

prompt - prompt to display to the user before they input a string which will be interpreted as if it was command-line input to the program
prompts (const char **) - An array of c-strings which will be displayed to the user in sequence as they input the values this program needs
prompts (function<string>**) - an array of functions whose values will be displayed to the user in sequence as they input the values this program needs
tests - an array of ints which correspond to the index of the test sequence which should run after each input. If not present, all tests will be reserved until after all input is provided
flagPrompt - a prompt to display to the user before they enter the string which will set all the flags. If not present, the user will not be able to set any flags through dynamic input.
Note: Currently, if the user fails to input the name of a flag where one is expected, that input is ignored (which will typically cause its arguments to be ignored as well). This behavior may change in the future.
flagHelpString - a prompt to display to the user if they input exactly "?" or "help" while setting flags. If not present, "?" and "help" don't do anything special there.

[edit] Exceptions

None

[edit] Notes

Only one test sequence can be set to run after each input. This is an imperfection that I decided to accept so that I could finish the library faster, and will be fixed on request.

If (1-2) is called, and the number of elements in prompts is less than the number of ordered vars, invokes undefined behavior.
If (1-2) is called, and the number of elements in prompts is greater than the number of ordered vars, the extra element(s) are ignored.

Macros CIP_strs(n) and CIP_strFs(n) can be used to create the array provided to prompts.

[edit] Examples

// input_handler ih is declared earlier
// with functions
ih.setupDynamicInput(CIP_strFs(3) {
    STR("create or apply (c/a)? "),
    STR("original file's name: "),
    TERNARY(creating, "new file's name: ", "transform file's name: ")},
    tests(1, 3, CIP::no_test),
    "input flags: ",
    "insert useful help message"
});

// with strings
ih.setupDynamicInput(CIP_strs(3) {
    "create or apply (c/a)? ",
    "original file's name: ",
    "other file's name: "}
});

// imitate cmd line
ih.setupDynamicInput("provide an argument list, as you would if you "
                     "were running this from cmd.exe: ");

Results: 

Case 1/2:
The first two prompts are always "create or apply (c/a)? " and "original file's name: ".
The last prompt is always "other file's name: " in case 2, but in case 1 is 
"new file's name: " if creating is true, and "transform file's name: " otherwise.

Case 1:
After the above, the final prompt "input flags: " is shown and the user gets an oppertunity
to do so. If they input "?" or "help", "insert useful help message" is printed, then
"input flags: " is displayed again and the user gets another chance to input flag values.

Case 3:
There is only one prompt, and the user's input to it is interpreted 
as a command-line argument list.
