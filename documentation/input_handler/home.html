CIP::input_handler

From cppreference.com
< cpp‎ | string
class input_handler;

input_handler offers a fairly comprehensive interface through which one can clearly and concisely program the code which processes the arguments which were passed to the program through the command line, with the option of having the class prompt the user for input if none are passed in that way.

All programs which use the CIP library will process input in a similar way. A comprehensive description of this class and how to use it can be found in this linked pdf, but here are the most important points about how it works:

  • There are (up to) two types of arguments, "ordered variables" and "flags"
  • flags are named variables which take a programmer-set number of arguments. They begin with "/theFlagsName" (where '/' is a delimiter set by the programmer)
  • Flag arguments are separated by spaces, or any other equivalent command-line delimiter.
  • Flags always take the same number of arguments every time they are set
  • Flags vars can be manually set to their default value by providing a string which you set for this purpose. Using this, it is possible to create flags which effectively take a non-constant number of arguments, where the last argument is followed by "some_string_which_denotes_end_of_arguments." It is currently impossible to set up a flag which can take a truely arbitrary number of arguments in this way.
  • The flags "/?" and "/help" always exist, and will cause the program's help string to be printed. The former always begins with '/'.
  • Ordered variables are arguments which are not flags. In the windows command "copy a b", "a" is passed to ordered var 1 and "b" is passed to ordered var 2.
  • The type of a given ordered var is always the same every time the program is run. Previous arguments currently cannot be used to inform what future arguments should be.
  • There is no designated time for flags to be input. Given a program with flags /n which accept n inputs each, the following are all equivalent argument lists (flags and their arguments are underlined, ordered vars are bold):
    • /0 /1 + /4 x y z * a b c d
    • a b c d /0 /1 + /4 x y z *
    • a b /4 x y z * /0 /1 + c d
    • a /4 x y z * b /0 c /1 + d
